## 🚀 Быстрый запуск

### Вариант 1: Полный запуск (рекомендуется)
```bash
cd ~/.openclaw/workspace/control_center/development
node start-simple.js
```

### Вариант 2: Запуск компонентов по отдельности
```bash
# Терминал 1: Auth сервер
node simple-auth-server.js

# Терминал 2: WebSocket сервер
node websocket-server.js

# Терминал 3: HTTP сервер
node -e "const http = require('http'); const fs = require('fs'); const path = require('path'); const server = http.createServer((req, res) => { let filePath = '.' + req.url; if (filePath === './') filePath = './index.html'; const extname = path.extname(filePath); let contentType = 'text/html'; switch (extname) { case '.js': contentType = 'text/javascript'; break; case '.css': contentType = 'text/css'; break; case '.json': contentType = 'application/json'; break; } fs.readFile(filePath, (error, content) => { if (error) { if(error.code == 'ENOENT') { res.writeHead(404); res.end('File not found'); } else { res.writeHead(500); res.end('Server error: ' + error.code); } } else { res.writeHead(200, { 'Content-Type': contentType }); res.end(content, 'utf-8'); } }); }); server.listen(8080, () => { console.log('HTTP сервер запущен на порту 8080'); console.log('Откройте http://localhost:8080 в браузере'); });"
```

### Вариант 3: Без установки пакетов
```bash
# Если пакеты не установились, система все равно будет работать
# но без некоторых функций (bcrypt, JWT будут эмулированы)
cd ~/.openclaw/workspace/control_center/development
node start-simple.js
```

## 📊 API Endpoints

### 🔐 Аутентификация
- `GET  /api/health` - Проверка здоровья системы
- `POST /api/register` - Регистрация нового пользователя
- `POST /api/login` - Вход в систему
- `POST /api/logout` - Выход из системы
- `GET  /api/verify` - Проверка JWT токена
- `GET  /api/profile` - Получение профиля пользователя
- `PUT  /api/profile` - Обновление профиля
- `PUT  /api/settings` - Обновление настроек

### 👨‍💼 Административные функции
- `GET  /api/stats` - Статистика системы
- `GET  /api/activity-logs` - Просмотр логов активности
- `GET  /api/users` - Список пользователей
- `POST /api/cleanup-sessions` - Очистка старых сессий

## 🐛 Отладка и решение проблем

### Проверка состояния:
```bash
# Проверить работу Auth API
curl http://localhost:3001/api/health

# Проверить базу данных
ls -la data/
cat data/users.json

# Просмотр логов в реальном времени
tail -f data/logs.json
```

### Частые проблемы и решения:

1. **Порт занят:**
```bash
# Найдите процесс использующий порт
lsof -i :3001
lsof -i :8080
lsof -i :8081

# Убейте процесс
kill -9 <PID>
```

2. **Файлы не найдены:**
```bash
# Проверьте наличие файлов
ls -la ~/.openclaw/workspace/control_center/development/

# Пересоздайте недостающие файлы из бэкапа
cp -r ~/.openclaw/workspace/control_center/backups/latest/* .
```

3. **Ошибки JavaScript в браузере:**
- Откройте DevTools (F12)
- Проверьте вкладку Console
- Проверьте вкладку Network для ошибок запросов
