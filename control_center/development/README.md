# 🚀 Development версия дашборда Атласа

Это development версия веб-дашборда для общения с ассистентом Атлас. Все изменения тестируются здесь перед переносом в production.

## 📁 Структура проекта

```
control_center/development/
├── index.html              # Основной HTML файл с чат-интерфейсом
├── script.js               # JavaScript логика (WebSocket, чат)
├── websocket-server.js     # WebSocket сервер для тестирования
├── backup-script.js        # Скрипт для автоматических бэкапов
├── package.json            # Зависимости Node.js
├── README.md               # Эта документация
└── (другие файлы)
```

## 🚀 Быстрый старт

### 1. Установка зависимостей
```bash
cd control_center/development
npm install
```

### 2. Запуск WebSocket сервера
```bash
npm start
# или с авто-перезагрузкой
npm run dev
```

### 3. Открыть в браузере
Откройте http://localhost:8081 в браузере.

## 🔧 Технические детали

### WebSocket сервер
- Порт: 8081
- Протокол: ws://localhost:8081
- Поддержка JSON сообщений
- Авто-реконнект при обрыве

### Бэкапы
Автоматическое создание резервных копий:
```bash
# Создать бэкап вручную
npm run backup

# Показать список бэкапов
npm run backup:list

# Автоматический режим (каждые 30 мин)
npm run backup:auto
```

Бэкапы хранятся в: `control_center/backups/YYYY-MM-DD/HH-MM-SS/`

## 🆘 Поиск проблем

### WebSocket не подключается
1. Проверьте, запущен ли сервер: `npm start`
2. Проверьте порт 8081: `curl http://localhost:8081`
3. Проверьте firewall: `sudo lsof -i :8081`

### Файлы не обновляются
1. Очистите кэш браузера (Ctrl+Shift+R)
2. Проверьте консоль браузера на ошибки
3. Перезапустите сервер: `npm run dev`

### Бэкапы не создаются
1. Проверьте права на запись в `control_center/backups/`
2. Проверьте наличие Node.js: `node --version`
3. Запустите с правами администратора если нужно
