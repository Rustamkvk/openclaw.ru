# Быстрый старт Контрольного центра Атласа

## 🚀 Запуск

1. **Откройте терминал** в папке Контрольного центра:
   ```bash
   cd ~/.openclaw/workspace/control_center/development
   ```

2. **Запустите Контрольный центр**:
   ```bash
   ./control-center-manager.sh start
   ```

   Или вручную:
   ```bash
   ./start-final-fixed.sh
   ```

## 🌐 Доступ к интерфейсу

1. **Откройте браузер** и перейдите по адресу:
   ```
   http://localhost:8082
   ```

2. **Войдите в систему** с учетными данными по умолчанию:
   - **Имя пользователя:** `admin`
   - **Пароль:** `Admin123!`

## 📊 Основные компоненты

### 🔐 Auth API (порт 3001)
- Регистрация и авторизация пользователей
- Управление сессиями
- Проверка здоровья системы

### 🔌 WebSocket Server (порт 8081)
- Чат с Атласом в реальном времени
- Уведомления и события
- Обновление данных в реальном времени

### 🌐 Frontend (порт 8082)
- Основной интерфейс Контрольного центра
- Управление памятью и индексами
- Мониторинг системы
- Чат с Атласом

## 🛠 Управление

### Запуск/остановка
```bash
# Запуск
./control-center-manager.sh start

# Остановка
./control-center-manager.sh stop

# Перезапуск
./control-center-manager.sh restart

# Статус
./control-center-manager.sh status

# Логи
./control-center-manager.sh logs
```

### Ручное управление процессами
```bash
# Запуск всех компонентов
./start-final-fixed.sh

# Остановка всех компонентов
pkill -f "simple-http-server-fixed.js"
pkill -f "simple-auth-no-deps.js"
pkill -f "websocket-server.js"
```

## 🔧 Решение проблем

### Проблема: "Порт уже занят"
Если порт 8080 занят системой:
- Контрольный центр автоматически использует порт 8082
- Все ссылки обновлены для работы с портом 8082

### Проблема: "Модуль express не найден"
Используется версия Auth Server без зависимостей:
- Не требует установки express
- Работает на встроенных модулях Node.js

### Проблема: "Окно авторизации не работает"
Проверьте:
1. Запущен ли Auth Server (порт 3001)
   ```bash
   curl http://localhost:3001/api/health
   ```
   
2. Доступен ли фронтенд (порт 8082)
   ```bash
   curl http://localhost:8082
   ```

3. Проверьте логи:
   ```bash
   ./control-center-manager.sh logs
   ```

## 📁 Структура файлов

```
control_center/development/
├── index.html              # Основной интерфейс
├── simple-http-server-fixed.js  # HTTP сервер (порт 8082)
├── simple-auth-no-deps.js  # Auth сервер без зависимостей (порт 3001)
├── websocket-server.js     # WebSocket сервер (порт 8081)
├── start-final-fixed.sh    # Скрипт запуска
├── control-center-manager.sh # Менеджер управления
├── auth-server.log         # Логи Auth сервера
├── websocket-server.log    # Логи WebSocket сервера
└── http-server.log         # Логи HTTP сервера
```

## 🔄 Автоматическое обновление

Контрольный центр автоматически:
- Обновляет индекс памяти каждые 15 минут
- Сохраняет логи в файлы
- Поддерживает актуальность данных

## 📞 Поддержка

Если возникли проблемы:

2. Перезапустите систему: `./control-center-manager.sh restart`
3. Обратитесь к Атласу за помощью
